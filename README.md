# PhishGuard AI - URL Phishing Detection System

A Spring Boot REST API backend that analyzes URLs for phishing indicators using rule-based heuristic scoring. Paste any URL and get back a verdict of **SAFE**, **SUSPICIOUS**, or **MALICIOUS** along with a risk score (0–100) and a detailed list of flagged signals. All scan results are persisted to a MySQL database.

## Project Structure

```
src/
└── main/
    └── java/
        └── com/phishguard/
            ├── PhishGuardApplication.java      # Entry point — starts embedded Tomcat on port 8080
            ├── controller/
            │   └── UrlController.java          # REST endpoints — receives and responds to HTTP requests
            ├── service/
            │   └── DetectionService.java       # Business logic — runs 10 heuristic detection rules
            ├── model/
            │   ├── AnalysisResult.java         # Response object — sent to client as JSON (not persisted)
            │   └── ScanResult.java             # Database entity — mapped to scan_results table
            └── repository/
                └── ScanResultRepository.java   # Spring Data JPA repository — zero SQL required
```

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/api/scan` | Analyze a URL — returns verdict, score, and flags |
| `GET` | `/api/history` | Retrieve all past scans (newest first) |
| `GET` | `/api/stats` | Get counts of total, malicious, suspicious, and safe scans |
| `GET` | `/api/health` | Health check — returns service status |
| `DELETE` | `/api/history/{id}` | Delete a single scan record by ID |
| `DELETE` | `/api/history` | Delete all scan records |

### POST `/api/scan` — Request Body

```json
{
  "url": "http://paypal-login.xyz/verify@account"
}
```

### POST `/api/scan` — Response

```json
{
  "verdict": "MALICIOUS",
  "score": 80,
  "flags": [
    "No HTTPS encryption",
    "Phishing keyword detected: 'paypal'",
    "High-risk TLD: .xyz",
    "@ symbol found (credential spoofing trick)",
    "Hyphen in domain name"
  ],
  "message": "High threat! Risk score: 80/100. Do NOT visit this URL."
}
```

## Detection Rules

The `DetectionService` runs up to 10 heuristic checks. Each rule adds points to the risk score (capped at 100).

| Rule | Points Added |
|---|---|
| No HTTPS | +20 |
| URL longer than 75 characters | +15 |
| `@` symbol in URL (credential spoofing) | +25 |
| IP address used instead of domain name | +30 |
| Phishing keyword found (`login`, `verify`, `paypal`, `amazon`, etc.) | +20 |
| High-risk TLD (`.tk`, `.xyz`, `.ru`, `.ml`, `.ga`, `.cf`, `.top`, etc.) | +25 |
| Hyphen in domain name | +10 |
| Excessive subdomains (3+ dots) | +20 |
| URL-encoded or obfuscated characters (`%`, `0x`) | +15 |
| Double slash in URL path | +10 |

### Verdict Thresholds

| Score | Verdict |
|---|---|
| 0 – 29 | SAFE |
| 30 – 59 | SUSPICIOUS |
| 60 – 100 | MALICIOUS |

## Setup & Running

### 1. Clone the repository

```bash
git clone https://github.com/your-username/phishguard.git
cd phishguard
```

### 2. Create the MySQL database

```sql
CREATE DATABASE phishguard;
```

### 3. Configure database credentials

Open `src/main/resources/application.properties` and update:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/phishguard?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=YOUR_MYSQL_PASSWORD
```

> Hibernate will automatically create the `scan_results` table on first run — no manual SQL needed.

### 4. Run the application

```bash
./mvnw spring-boot:run
```

The server starts at `http://localhost:8080`.

Confirm it's running:

```bash
curl http://localhost:8080/api/health
```

Expected response:
```json
{
  "status": "UP",
  "service": "PhishGuard AI"
}
```

## Database Schema

The `scan_results` table is auto-generated by Hibernate based on `ScanResult.java`:

| Column | Type | Description |
|---|---|---|
| `id` | BIGINT (PK, AUTO INCREMENT) | Unique scan ID |
| `url` | TEXT | The URL that was scanned |
| `verdict` | VARCHAR | MALICIOUS / SUSPICIOUS / SAFE |
| `risk_score` | INT | Score from 0 to 100 |
| `flagged_signals` | TEXT | Comma-separated list of triggered rules |
| `scanned_at` | DATETIME | Timestamp of the scan |

## Key Design Decisions

- **AnalysisResult vs ScanResult** — `AnalysisResult` is a transient response object (flags stored as `List<String>` for clean JSON). `ScanResult` is the persisted entity (flags stored as a comma-separated `String` since MySQL cannot store a Java List directly).
- **No SQL written** — All database operations go through `ScanResultRepository` which extends `JpaRepository`. Spring Data JPA generates queries from method names automatically.
- **CORS enabled** — `@CrossOrigin(origins = "*")` on the controller allows any frontend origin to call the API.
- **Rule-based detection** — No ML model; heuristic scoring is fast, explainable, and requires no training data.
